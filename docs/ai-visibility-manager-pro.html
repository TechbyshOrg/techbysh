Welcome & Overview

AI Visibility Manager Pro is a standalone, performance-engineered WordPress plugin designed for enterprise-level search engine optimization, bot firewall defense, structured entity linking, and WooCommerce sales attribution in the era of artificial intelligence crawlers.

Unlike generic SEO utilities, this plugin is tailored specifically to interface with LLM crawler engines, including OpenAI's GPTBot, Anthropic's ClaudeBot, and search models from Perplexity, Cohere, Meta, and Google. It provides systems administrators and developers with total control over how artificial intelligence discovers, reads, indexes, and attributes traffic to a WordPress environment.

Crawl Control PHP Hook

Prevent CPU spikes and scrape overload by filtering, rate-limiting, or challenging search crawlers on initial loading hooks before expensive templates load.

Markdown Export Virtual Router

Serve custom-cached /llms.txt directories and full-page markdown logs (/llms-full.txt) specifically structured for training corpus ingestion.

Attribution Engine Cookies

Track purchase journeys and store visitor paths from AI referral sources (Perplexity, ChatGPT, Claude) with optimized data structures and database models.

System Architecture

The plugin relies on an Object-Oriented singleton architecture (instantiated as AIVM_Pro_Plugin) and implements core hook bindings dynamically. This ensures that the codebase maintains a minimal memory footprint and zero external library dependencies.

GDPR and Privacy Policy: In compliance with global data privacy requirements, the plugin never stores raw visitor IP addresses. All incoming request IPs are processed using a secure, one-way hash algorithm (md5($ip)) prior to database logging or transient generation.

Database assets are safely isolated via standard WordPress prefix mapping (using $wpdb->prefix), ensuring compatibility with multi-site networks and custom prefixes.

Real-Time AI Firewall & Limits

The firewall acts at the early plugins_loaded hook, executing before WordPress does template lookups or database routing. This prevents resource starvation when aggressive AI scrapers recursively request pages.

For each recognized crawler, you can apply four distinct policies:

  • Allow: Passes the request through to the standard rendering pipeline.
  • Block: Halts execution immediately with a 403 Forbidden response.
  • Rate Limit: Imposes request limits over minute, hour, and day windows. If exceeded, returns a 429 Too Many Requests with a Retry-After: 60 header. Limit counts are stored securely using WordPress Transients.
  • Challenge: Renders a client-side JavaScript verification screen. Normal browsers automatically solve the challenge and receive a bypass cookie, while headless scraper scripts are locked out with a 401 Unauthorized response.

llms.txt & llms-full.txt Catalogs

To allow search indexers and AI agents to quickly summarize your site structure and parse content, the plugin registers virtual rewrite routes to serve markdown files:

  • /llms.txt: Displays a high-level site summary followed by a clean markdown list containing up to 250 of the most recently published URLs.
  • /llms-full.txt: Builds a combined, lightweight document exporting full posts, pages, and products in a clean, human-readable markdown format.

The parsing pipeline converts complex HTML pages into clean markdown elements dynamically using regex replacement patterns, removing scripts, block comments, and custom HTML attributes.

HTML to Markdown Parser Logic (class-aivmp-llms-txt.php) 
public function html_to_markdown( $html ) {
    // Strip shortcodes and Gutenberg blocks comments
    $html = strip_shortcodes( $html );
    $html = preg_replace( '/<!--\s+\/?wp:[^>]+-->/', '', $html );

    // Strip script blocks and stylesheets
    $html = preg_replace( '/<(script|style)[^>]*?>.*?<\/\1>/is', '', $html );

    // Format headings into markdown markers
    $html = preg_replace( '/<h1[^>]*?>(.*?)<\/h1>/i', "\n# $1\n", $html );
    $html = preg_replace( '/<h2[^>]*?>(.*?)<\/h2>/i', "\n## $1\n", $html );

    // Strip all remaining HTML tags
    $html = wp_strip_all_tags( $html );
    return trim( $html );
}

Performance Notice: Content caching is highly recommended. Set the cache duration setting (aivmp_llms_cache_duration) to keep the generated markdown cached in transients. This avoids reprocessing heavy queries on concurrent visits.

AI XML Sitemap

In addition to markdown indexes, the plugin publishes a virtual index map at /ai-sitemap.xml. This sitemap aggregates site content metadata and organizes links for rapid crawler indexing.

The sitemap engine automatically queries and structures the following elements:

  • Core Content: Singular posts and pages with last modified timestamps.
  • WooCommerce Integration: Appends products with tailored crawl priorities (0.8) and update frequencies.
  • Taxonomy Terms: Catalogs categories, tags, and product categories.
  • Authors: Lists profile archive routes for content publishers on the platform.

To prevent database lockups during concurrent crawls, sitemap results are compressed and stored in the aivmp_sitemap_cache transient for 24 hours.

WooCommerce AI Attribution

The attribution module monitors incoming referrals to track customers originating from AI platforms. It records paths and calculates conversion value on checkout.

Tracking Strategy: The customer landing page and referring source are kept in active PHP sessions and cookie variables (aivmp_wc_referrer and aivmp_wc_landing_page). The customer’s relative browsing history is stored in a cookie array limited to 15 entries to prevent cookie header size overflow.

When a customer completes an order, the woocommerce_checkout_order_processed hook captures the transaction details and persists them inside the wp_aivmp_wc_attribution table:

  • Referrer: The matched AI search platform (ChatGPT, Perplexity, Claude, Gemini, etc.).
  • Landing URL: The customer entry point on the website.
  • Order Total: Total cart value, including tax and shipping.
  • Journey Log: List of relative pages visited before checking out.

Analytics, Reports & Cron

Admin dashboard interfaces display interactive crawl metrics, block tallies, and sales referral trends. The charts are powered by a localized version of Chart.js v3.9.1, keeping all asset loads offline and GDPR-compliant.

Scheduled Executive Reports: If enabled, a WordPress cron hook (aivmp_send_scheduled_reports) triggers on a regular interval (daily, weekly, or monthly) to compile traffic analytics, sales conversions, and source distributions, emailing them to the configured administrator address.

Exporting Data: Developers and managers can navigate to the Tools tab to run manual reporting. The plugin generates structured CSV dumps of database records or triggers print-ready HTML-to-PDF styled stylesheets for executive presentations.

Server Configurations & Rewrites

WordPress handles virtual routes automatically by appending rules to the .htaccess configuration on Apache servers. However, on custom stacks or Nginx environments, manual directives may be required.

Nginx Server Block Directives 
# AI Visibility Manager Pro Virtual Routes
rewrite ^/llms\.txt$ /index.php?aivmp_llms_txt=1 last;
rewrite ^/llms-full\.txt$ /index.php?aivmp_llms_full_txt=1 last;
rewrite ^/ai-sitemap\.xml$ /index.php?aivmp_sitemap=1 last;

# Default fallback rule for general WP permalinks
location / {
    try_files $uri $uri/ /index.php?$args;
}
Apache Rewrite Directives 
<IfModule mod_rewrite.c>
RewriteEngine On
RewriteBase /
RewriteRule ^llms\.txt$ index.php?aivmp_llms_txt=1 [L,QSA]
RewriteRule ^llms-full\.txt$ index.php?aivmp_llms_full_txt=1 [L,QSA]
RewriteRule ^ai-sitemap\.xml$ index.php?aivmp_sitemap=1 [L,QSA]
</IfModule>

FAQ & Diagnostics

Q: The virtual routes (/llms.txt, /ai-sitemap.xml) are returning 404 errors.

A: This typically indicates that rewrite rules have not been flushed. Go to Settings > Permalinks in the WordPress dashboard and click Save Changes. This triggers a core rewrite rules rebuild. Also, ensure your site does not use plain permalinks (e.g. ?p=123).

Q: Does the firewall restrict standard Search Engine crawlers like Googlebot or Bingbot?

A: No. The firewall strictly maps specifically flagged AI-centric agents (such as GPTBot, ClaudeBot, and CCBot). General web crawlers used for standard search result listings are not affected by default.

Q: How can I manually trigger a rebuild of cached sitemaps or markdown catalogs?

A: The cache is flushed automatically whenever posts are updated or saved. To manually clear the cache, go to the Status tab in the plugin settings and click the Flush Transients Cache button.

Q: Where are the local scripts loaded from?

A: To maintain compliance with WordPress.org guidelines regarding data leakage and CDNs, Chart.js is bundled locally inside /admin/js/chart.min.js. No remote scripts or styles are requested during operation.

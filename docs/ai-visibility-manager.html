AI Visibility Manager Documentation

Complete technical reference and installation guide for controlling search crawlers and Large Language Models (LLMs).

1. Introduction

AI Visibility Manager is a WordPress plugin designed to help webmasters manage how Artificial Intelligence bots, search LLMs, and conversational engines index their sites. In the modern web environment, new crawlers scan sites to feed training datasets and compile references for conversational search boxes (like ChatGPT Search or Perplexity).

Why this matters: Standard robots.txt rules do not provide granular control over the latest AI bots, nor do they track referred conversational traffic. This plugin bridges that gap in a lightweight, self-contained way.

The plugin operates on three primary pillars:

  • Dynamic Bot Control: Filter robots.txt dynamically to disallow specific crawler tokens.
  • Markdown Indexing: Generate dynamic, cached llms.txt standards.
  • Traffic Monitoring: Safe local logging of referrer agents without bloating DB tables.

2. Installation & Quick Start

To install the plugin manually or through the WordPress admin panel, follow these steps:

  1. Download the plugin .zip package from the official directory.
  2. In your WordPress dashboard, navigate to Plugins > Add New and click Upload Plugin.
  3. Choose the downloaded file and click Install Now.
  4. Once installed, click Activate Plugin.
  5. Go to Settings > AI Visibility Manager in your admin menu to adjust configurations.
Requirement: The plugin requires Pretty Permalinks (e.g. Post Name) to be enabled under Settings > Permalinks. Dynamic virtual routing for the /llms.txt file will fail if plain query permalinks are active.

3. Overview Dashboard

The Overview panel is divided into three sections to help you monitor your site's discoverability:

A. Diagnostics Widgets

Displays at-a-glance metrics including Total AI Referrals, details of the Last Referral (e.g., bot name and how long ago it arrived), and reachability indicators for the /llms.txt and /robots.txt files.

B. Automated Health Check

Runs local tests to check settings without calling external APIs. The checks inspect whether:

  • Pretty permalinks are active.
  • The dynamic /llms.txt file is reachable.
  • The /robots.txt file is reachable and not blocked by physical root files.
  • Your site visibility settings encourage or discourage search indexing.
  • The site is in maintenance mode.

C. Content Discoverability Report

Shows the count of public posts and pages currently indexed for LLM ingestion, count of non-public excluded posts, and the exact timestamp when the transient cache was last rebuilt.

5. Dynamic llms.txt Generator

The llms.txt standard provides structured markdown summaries specifically optimized for LLMs. The plugin dynamically generates this output under the URL https://yourdomain.com/llms.txt.

The file structure generated matches this format:

# Site Name

## About
Site description details.

## Recent Content
- [Post Title 1](https://yourdomain.com/post-1/)
- [Page Title 2](https://yourdomain.com/page-2/)
...
Performance Caching: To prevent resource usage, the dynamic output is cached using WordPress transients for up to one week. The cache is automatically cleared whenever you publish, edit, or trash posts or pages.

6. AI Referrals Analytics

The plugin tracks incoming visits referred by conversational search engines by parsing the HTTP Referrer header. It captures referrers from ChatGPT, Perplexity, Claude, Gemini, Copilot, and Bard.

Transient Debounce Check

To protect your database from spam and resource usage, the referral logger hashes the visitor's IP address, requested destination URL, and referrer source. It sets a transient key for 5 minutes. If the same visitor clicks another link or refreshes the page within this window, the visit is debounced and not saved to the database.

You can view top referral sources, top pages visited by AI referrals, and download a CSV file of the logged spreadsheet from the Tools panel.

7. Diagnostics & Troubleshooting

Problem 1: llms.txt returns a 404 Page Not Found error

Reason: Your permalink structure is set to "Plain", or WordPress rewrite rules have not flushed correctly.

Solution: Navigate to Settings > Permalinks, change the setting to Post Name, and save. Then go to Settings > AI Visibility Manager > Tools and click Rebuild Cache to flush rules.

Problem 2: robots.txt doesn't show the blocked AI bot rules

Reason: You have a physical robots.txt file in your site's root directory, which prevents WordPress from generating dynamic virtual rules.

Solution: Connect to your site via FTP or File Manager and delete the physical robots.txt file from the root folder. WordPress will then handle robots.txt requests dynamically.

8. GDPR & Privacy Compliance

This plugin is built with GDPR and user privacy in mind:

  • No user tracking data or logs leave your local server.
  • The plugin does not write raw IP addresses to the database.
  • IP addresses are hashed using MD5 during the transient check to prevent duplicate logs and are then discarded.

9. Developer Hooks & Filters

Developers can customize the plugin's behavior using filters:

A. Customize llms.txt output

If you want to alter the Dynamic Markdown content before it is output or cached:

add_filter( 'aivm_llms_txt_output', 'custom_llms_txt_content' );
function custom_llms_txt_content( $output ) {
    // Append custom legal or about info at the bottom
    $output .= "\n## Custom Index\n- [Terms](https://yoursite.com/terms/)\n";
    return $output;
}

B. Add custom AI bots to block in robots.txt

To block additional crawlers not listed by default in the free control panel:

add_filter( 'aivm_custom_robots_rules', 'add_custom_bots' );
function add_custom_bots( $rules ) {
    $rules .= "\nUser-agent: SpecializedBot\nDisallow: /\n";
    return $rules;
}
